The Importance of XHTML 1.0 Strict Validation

What is XTML 1.0 Validation?

XHTML 1.0 Validation is a quality confirmation program used to look at Hypertext Markup Language (HTML) markup components for syntax errors. A validator can be a valuable device for a HTML client who gets information electronically from an assortment of information sources.

How it works

Head to this link validator or search up HTML validator up in your web browser and click on the first link. The page should look something like this



Next click on the third tab that says, "Validate by Direct Input". The page should look something like this:



This essentially means that you just copy paste your entire code into the box on the new screen. This will be easier incase the platform that you run it on has some internal errors and displays bugs that have nothing to do with your code (i.e edstem). After copy pasting your code, click the check button and wait for the results to render. Either the screen will turn up red and state where the issue is (what line of code and which column the problem is) or the screen will turn green. Let's look at some example of errors.

Validation Errors

Some errors the validator will pick up can be forgetting a closing tag, invalid syntax for a comment, not allowing an element in a specific place, for example the style tag in the body. It should instead be used in the head and so on. You can look at the full list at this website, Validator Errors Explained
